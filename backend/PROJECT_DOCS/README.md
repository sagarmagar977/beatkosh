# BeatKosh Project Docs

This folder contains consolidated documentation for the current BeatKosh implementation and the Beat22 alignment work.

## Files

- `PROJECT_DOCS/PRD_BEAT22_ALIGNMENT.md`: Full product + technical documentation for frontend and backend.
- `PROJECT_DOCS/MASTER_EXECUTION_PLAN.md`: Unified execution plan combining original PRD scope and Beat22 references.
- `PROJECT_DOCS/IMPLEMENTATION_TRACKER.md`: Live tracker with `COMPLETED / PARTIAL / NOT_STARTED / BLOCKED`.
- `PROJECT_DOCS/SESSION_HANDOFF_CONTEXT.md`: Resume guide for next sessions (what to run, what to do next).
- `PROJECT_DOCS/scripts/run_checks.ps1`: Automation script to run backend/frontend validation checks.
- `PROJECT_DOCS/AUTOMATION_REPORT.md`: Latest generated validation report.
- `PROJECT_DOCS/RELEASE_RUNBOOK.md`: Release checklist, verification flow, and rollback notes.
- `.github/workflows/ci.yml`: CI pipeline for backend tests and frontend lint/type-check.

## Scope

- Marketplace (beats, bundles, tapes)
- Artist and producer role flows
- Orders, payments, wallet, verification
- Collaboration projects and milestones
- Beat22 reference alignment layer (API + UI)
